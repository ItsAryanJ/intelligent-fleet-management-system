06RISKS & MITIGATIONS
HOW THIS ROADMAP WAS BUILT
Twelve agents each read the actual source at HEAD d36ea82 and produced concrete, file-level
improvement plans across the four modules, auth, code quality, UI/UX, docs, honesty and stretch; an
adversarial critic then pressure-tested whether executing all of it clears 9.5 under the exact rubric
weights and corrected the cross-dimension sequencing; a synthesis pass built the scoring model and
phased plan. Effort sizes: S < half a day, M ~1–2 days, L ~3–5 days. Scores are calibrated to the same
MVP-depth Lite-PRD bar used in the v1→v2→v3 evaluations. The plan deliberately favours wiring and
fixing existing backend work over new build — which is why +2.0 is realistic in roughly four focused
weeks.
NCRTC BMS — Roadmap to 9.5+ · Companion to Evaluation v3 7.6 → 9.6 target · Generated 7 July 2026The geofence contradiction is the single biggest swing lever. If the radius-circle path ships it
stays DENIED (it is exactly what a grader rejected as "wrong condition / orphaned") and the frontend
gets wired to the wrong contract. Mitigation: reconcile to the seeded-POLYGON + ST_Contains +
exit-breach path before building the banner/polygon consumer.
No owner for Auth/RBAC end-to-end caps Functional at ~9.2–9.3 (Auth is a 7.5 component of the
40% axis). Mitigation: the dedicated P0/P1 Auth track; if it slips, overall still clears 9.5 on stretch +
Docs + Honesty but with thin margin.
New CMS columns + the geofence polygon rely on runtime create_all and are in no migration.
Flipping the Dockerfile to alembic upgrade head before the baseline captures them boots a fresh
clone without those columns. Mitigation: author the baseline AFTER those columns land, and keep
create_all as the Docker path until the migration is proven on a fresh PostGIS clone.
The 52-site depot-scope refactor is high regression risk across 13 routers immediately pre-
demo — and it is the mechanism that fixes the routes leak. Mitigation: land the RED-first depot-
scoping regression test before the refactor and gate the refactor behind it; do not ship blind.
The demo video is the tail dependency of the entire programme — its CMS climax and
geofence/SLA beats require every module fix landed. Mitigation: record LAST, treat "no dead-end"
as its acceptance test, keep P0 CMS-publish + P1 read-receipt on the critical path.
Alembic Geometry-column autogenerate routinely produces a non-applying baseline
(geoalchemy2 pulls in spatial system tables) — effort is realistically L, not M. Mitigation: hand-edit
the baseline to CREATE EXTENSION postgis first and verify on an empty PostGIS DB before touching
the Dockerfile.
AVLS real on-time/late detection depends on Scheduling's planned-arrival offsets. If that link is
never wired, both ship an honest speed-only heuristic and each leaves ~0.4 on the table. Mitigation:
sequence the stop-editor before AVLS operational colouring, or explicitly label "Idle/Moving" to
preserve honesty.
